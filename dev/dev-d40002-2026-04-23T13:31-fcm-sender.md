## Goal
Implement FCM v1 HTTP API sender. Worker signs a JWT using the Firebase service account, exchanges for an access token (cached in KV with TTL), and POSTs to `fcm.googleapis.com/v1/projects/<id>/messages:send`.

## Acceptance criteria
- [ ] User creates Firebase project (MANUAL.md item filed at start of epic)
- [ ] Service account JSON stored as Worker secret `FCM_SA_JSON`
- [ ] `fcm.ts` implements: RS256 JWT signing (via `crypto.subtle`), token exchange, caching in KV
- [ ] `sendToToken(token, payload)` POSTs to FCM v1 endpoint with bearer token
- [ ] Returns success/failure; failures logged to KV dead-letter
- [ ] Integration test (manual) delivers a "hello from devx" push to a test FCM token
- [ ] Unit tests cover JWT signing + cache hit/miss

## Technical notes
- Cache access token for 55 minutes (FCM tokens last 1h; leave 5min buffer)
- Dead-letter key: `dlq:<timestamp>:<token>`, TTL 7 days
- If the FCM service account JSON is malformed, Worker returns 500 with clear error
