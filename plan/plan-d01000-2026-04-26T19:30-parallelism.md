## Goal

Run N=3 workers in parallel without rip-through. Capacity caps, permission enforcement, and the rip-through repro test all turn green.

## Scope

Five epics from [`ROADMAP.md § Phase 3`](../docs/ROADMAP.md#phase-3--parallelism--coordination-week-4):

- `epic-locks` — `manager.lock`, `concierge.lock`, `spec-<hash>.lock`, `ci-wait-<branch>.lock` via O_EXCL.
- `epic-intents` — `.devx-cache/intents/<agent-id>.json` read by Manager pre-spawn.
- `epic-rip-through-test` — integration test that races two `/devx` against the same branch family.
- `epic-capacity-management` — priority tiers + `capacity.{max_concurrent,usage_cap_pct,daily_spend_cap_usd}`.
- `epic-permission-enforcement` — `permissions.bash.{allow,ask,deny}` wired into worker spawn; `.env`/SSH hard-denied regardless of config.
